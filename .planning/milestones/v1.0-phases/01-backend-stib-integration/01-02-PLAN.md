Create `DeparturesService.java` in `com.transit.service`:
- Annotate with `@Service`
- Constructor-inject `StibApiClient` and `Clock` (use `Clock.systemDefaultZone()` bean or inject `Clock` directly — create a `@Bean Clock clock() { return Clock.systemDefaultZone(); }` in a config class if needed, or just use `Clock.systemDefaultZone()` as default in the constructor)
- Define hardcoded route constants as a record or inner class:
  ```
  WOEST: pointId="5008", stopName="Woest", lineId="51", direction="Gare du Midi"
  PANNENHUIS: pointId="8784", stopName="Pannenhuis", lineId="6", direction="Elisabeth"
  ```
- Method `getDepartures()` returns the generated `DeparturesResponse` model:
  1. Call `stibApiClient.fetchWaitingTimes(List.of("5008", "8784"))`
  2. For each hardcoded route, find matching results by pointId AND lineId from the STIB response
  3. For each matching PassingTime, compute `minutesUntilArrival = (int) MINUTES.between(now, expectedArrivalTime) + 1` (round up, using injected Clock)
  4. Only include PassingTime entries where the destination matches the commute direction (compare destination.fr() case-insensitively with the route's direction — e.g., "ELISABETH" matches "Elisabeth")
  5. Build RouteDepartures for each route, set lastUpdated to OffsetDateTime.now(clock)
  6. If StibApiClient throws, catch the exception and rethrow as a `ResponseStatusException(HttpStatus.BAD_GATEWAY, "STIB API unavailable")` or let the controller handle it

Create `DeparturesApiController.java` in `com.transit.controller`:
- Annotate with `@RestController`
- Implement the generated `DeparturesApi` interface (exact interface name depends on codegen — read the generated code after Plan 01 or check build/generated)
- Constructor-inject `DeparturesService`
- Implement the GET handler:
  - Call `departuresService.getDepartures()`
  - Return `ResponseEntity.ok(response)`
  - Catch exceptions from the service and return 502 with ErrorResponse(error="STIB_API_ERROR", message=exception.getMessage())

Use `@ExceptionHandler` on the controller or a try-catch in the method to map STIB failures to 502.

- Test case 3: "filters departures by direction" — mock a result with passingtimes going to BOTH directions (e.g., Elisabeth and Roi Baudouin for line 6). Assert only Elisabeth departures appear in the Pannenhuis route.
