## Directory Purposes

**api-spec/:**
- Purpose: OpenAPI contract (single source of truth)
- Contains: `openapi.yaml` defining all endpoints, schemas, error responses
- Key files: `openapi.yaml` is regenerated when API changes

**backend/:**
- Purpose: Java Spring Boot backend application
- Build: `./gradlew build` compiles, generates OpenAPI stubs, runs tests
- Key files:
  - `build.gradle` - OpenAPI Generator plugin configured to generate from `../api-spec/openapi.yaml`
  - `TransitApplication.java` - Spring Boot entry point
  - `RoutesApiController.java` - Implements generated `RoutesApi` interface

**backend/src/main/java/com/transit/:**
- Purpose: All Java source code
- Current structure (MVP):
  - `controller/` - REST API handlers
  - `service/` - Business logic (currently mixed with infrastructure)
  - `config/` - Spring configuration
- Target structure (from ARCHITECTURE-DECISIONS.md):
  - `domain/` - Pure domain logic (planned)
  - `application/` - Use cases and ports (planned)
  - `infrastructure/` - Adapters (planned)

**backend/build/generated/:**
- Purpose: Read-only code generated by OpenAPI Generator
- Contains: `RoutesApi.java` interface and model DTOs
- Rule: Never edit generated code; changes go through `api-spec/openapi.yaml`

**frontend/:**
- Purpose: React TypeScript frontend application
- Build: `npm run dev` (Vite dev server), `npm run build` (production build)
- Key file: `vite.config.ts` with Tailwind CSS v4 integration

**frontend/src/:**
- Purpose: All TypeScript/React source code
- Structure follows Clean Architecture with layers:
  - `entities/` - Domain types (pure TypeScript, no React)
  - `use-cases/` - Business logic (TS classes, no React)
  - `infrastructure/` - Technical implementations (API clients, DI, mocks)
  - `presentation/` - React components, pages, hooks
  - `gen/` - OpenAPI-generated types and hooks (read-only)

**frontend/src/entities/:**
- Purpose: Pure TypeScript domain entities
- Contains: Type definitions matching backend domain model
- Current: `dashboard.ts` with DashboardData, FavoriteRoute, etc.
- Rule: No React imports, no external dependencies (pure TypeScript)

**frontend/src/use-cases/:**
- Purpose: Application business logic classes
- Structure:
  - `ports/` - Interfaces defining external dependencies
  - `dashboard/` - Dashboard use cases (GetDashboard)
  - `routes/` - Route use cases (planned for Feature 2)
- Current: `GetDashboard` demonstrates pattern
- Rule: Use cases depend on ports (interfaces), not implementations

**frontend/src/use-cases/ports/:**
- Purpose: Dependency inversion — define contracts for external systems
- Contains: Interfaces like `IDashboardService`
- Current: `dashboard-service.ts` interface
- Rule: Infrastructure implements these ports

**frontend/src/infrastructure/:**
- Purpose: Technical implementations and framework setup
- Subdirectories:
  - `di/` - React Context-based dependency injection
  - `mock/` - Mock implementations for testing and development
  - `api/` - API client wrappers (planned for Feature 2)
- Current: `ServiceProvider` wraps app with mocked dashboard service

**frontend/src/infrastructure/di/:**
- Purpose: Dependency injection container via React Context
- Components:
  - `service-context.ts` - Context definition
  - `service-provider.tsx` - Provider component wrapping App
  - `use-dashboard-service.ts` - Custom hook to access context
- Pattern: `<ServiceProvider service={implementation}><App /></ServiceProvider>`

**frontend/src/presentation/:**
- Purpose: React UI layer
- Subdirectories:
  - `pages/` - Full-page components (DashboardPage)
  - `components/` - Reusable UI components
  - `hooks/` - Custom hooks wrapping use cases
  - `utils/` - Presentation utilities (greeting logic, formatters)
- Current: Dashboard page with header, greeting, favorite route card, departures

**frontend/src/gen/:**
- Purpose: Read-only OpenAPI-generated code
- Generated by: Kubb (TypeScript API client generator)
- Contains:
  - `types/` - TypeScript DTOs matching OpenAPI schemas
  - `hooks/` - TanStack Query hooks for queries/mutations
  - `schemas/` - Zod validators
- Rule: Never edit; regenerate from `api-spec/openapi.yaml` via `npm run kubb:generate`

## Key File Locations

**Entry Points:**

- API Contract: `api-spec/openapi.yaml`

**Core Logic:**
- Backend Routes: `backend/src/main/java/com/transit/service/RouteService.java`

**Testing:**
- Backend: `backend/src/test/java/` (not yet created)
- Frontend: `frontend/src/**/*.test.ts` (e.g., `frontend/src/use-cases/dashboard/get-dashboard.test.ts`)

## Naming Conventions

**Backend Files:**
- Classes: `PascalCase.java` (e.g., `RoutesApiController.java`, `RouteService.java`)
- Packages: `lowercase.dot.separated` (e.g., `com.transit.controller`)
- Generated: Keep in `build/generated/` directory

**Frontend Files:**
- Components: `PascalCase.tsx` (e.g., `DashboardPage.tsx`, `FavoriteRouteCard.tsx`)
- Hooks: `camelCase.ts` or `useCamelCase.ts` (e.g., `useDashboard.ts`, `greeting.ts`)
- Utilities: `kebab-case.ts` or `camelCase.ts` (e.g., `dashboard-service.ts`, `greeting.ts`)
- Types/Entities: `camelCase.ts` (e.g., `dashboard.ts`)
- Tests: `filename.test.ts` (e.g., `get-dashboard.test.ts`)

**Backend Naming Patterns:**
- Aggregates: `User`, `SavedRoute`
- Entities: `TransportOption`
- Value Objects: `Location`, `Stop`, `Line`, `Direction`, `Email`
- Repositories (interfaces): `UserRepository`, `SavedRouteRepository`
- Services (use cases): `CreateSavedRouteUseCase`, `GetUserRoutesUseCase`
- Controllers: `RoutesApiController`, `UsersApiController`
- Adapters: `StibTransitAdapter`, `ClerkAuthAdapter`, `JpaSavedRouteRepository`

**Frontend Naming Patterns:**
- Pages: `*Page.tsx` (e.g., `DashboardPage.tsx`, `RoutesPage.tsx`)
- Components: `*Card.tsx`, `*Header.tsx`, `*List.tsx` (e.g., `FavoriteRouteCard.tsx`)
- Hooks: `use*` (e.g., `useDashboard.ts`, `useAuth.ts`)
- Entities/Types: `domain-concept.ts` (e.g., `dashboard.ts`, `route.ts`)
- Use Cases: `get-*.ts`, `create-*.ts`, `update-*.ts` (e.g., `get-dashboard.ts`)
- Port Interfaces: `*-service.ts` (e.g., `dashboard-service.ts`)
- Implementations: `*-service-impl.ts` or in `infrastructure/api/` (e.g., `api-dashboard-service.ts`)

## Where to Add New Code

**New Backend Feature (e.g., User Management):**
- Create API endpoints:
  1. Add paths to `api-spec/openapi.yaml` (e.g., `/api/users`, `/api/users/me`)
  2. Run `./gradlew compileJava` to generate interfaces
  3. Implement controller in `backend/src/main/java/com/transit/controller/UsersApiController.java`
  4. Implement service in `backend/src/main/java/com/transit/service/UserService.java`
- Write tests: `backend/src/test/java/com/transit/service/UserServiceTest.java`
- Current MVP approach: Service uses in-memory storage (ConcurrentHashMap)

**New Frontend Component:**
- Create in `frontend/src/presentation/components/` with `PascalCase.tsx`
- If component needs data:
  1. Add use case to `frontend/src/use-cases/` if not exists
  2. Create custom hook in `frontend/src/presentation/hooks/use*.ts` that wraps use case
  3. Component calls hook to get state
- Follow Dark Mode pattern: Check `useDarkMode()` in component for conditional styling

**New Frontend Page:**
- Create in `frontend/src/presentation/pages/*Page.tsx`
- Import page in `App.tsx` and add routing (TanStack Router planned for Feature 3)
- For data: Create use case and hook, use in page component

**New Use Case (Frontend or Backend):**
- Frontend: Create class in `use-cases/{feature}/` that accepts port interface in constructor
- Backend (future): Create class in `application/{feature}/` with same pattern
- Define port interface in `use-cases/ports/` if new external dependency needed
- Example: `class CreateRoute { constructor(private readonly routeRepo: IRouteRepository) {} execute(cmd) { ... } }`

**Shared Utilities:**
- Frontend presentation utils: `frontend/src/presentation/utils/`
- Frontend domain validators: `frontend/src/entities/validators/` (planned)
- Backend domain services: `backend/src/main/java/com/transit/domain/services/` (planned)

**Testing:**
- Backend unit tests: `backend/src/test/java/com/transit/{layer}/{filename}Test.java`
- Frontend unit tests: `frontend/src/{layer}/**/*.test.ts` (co-located with source)
- Current: `frontend/src/use-cases/dashboard/get-dashboard.test.ts` demonstrates pattern

## Special Directories

**backend/build/generated/:**
- Purpose: OpenAPI Generator output
- Generated: Yes (via `./gradlew compileJava`)
- Committed: No (git-ignored)
- Rule: Never edit manually — regenerate via Gradle

**frontend/src/gen/:**
- Purpose: Kubb-generated OpenAPI client
- Generated: Yes (via `npm run kubb:generate`)
- Committed: Yes (source of generated types, hooks)
- Rule: Regenerate when API spec changes, commit for reproducibility

**frontend/node_modules/:**
- Purpose: npm dependencies
- Generated: Yes (via `npm install`)
- Committed: No (git-ignored)

**backend/build/ and backend/.gradle/:**
- Purpose: Gradle build cache and outputs
- Generated: Yes
- Committed: No (git-ignored)
